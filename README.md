# News Radar Dashboard

Read-only editor dashboard for the [News Radar](https://github.com/HsinTiger/news-radar) automated publishing pipeline. Six pages (Home / Queue / Archive / Dropped / Persona / Settings) rendered from a SQLite database committed to the `state` branch of the backend repo; dashboard refreshes every 5 minutes via sql.js running fully in the browser.

**Architecture decisions:** read-only by design — any write (approve / edit / retry) is done in the backend. No server, no auth, deploys as a static site to GitHub Pages. Data lives in `HsinTiger/news-radar@state:data/01_harvest/news_radar.db` and is fetched directly as raw bytes.

---

## Getting started

```bash
npm install
npm run dev           # http://localhost:5173/news-radar-dashboard/   (mock data)
npm run dev:live      # same URL, hits the real state-branch DB via sql.js
```

Mock mode (default) uses `src/data/mockData.js`, pinned to 2026-04-23T06:30:00Z for stable screenshots and offline work. Live mode swaps in `src/lib/dbAdapter.js`, which reads the sqlite file directly in the browser and polls every 5 minutes.

Build:

```bash
npm run build         # mock-mode build (for local preview)
npm run build:live    # VITE_USE_LIVE_DB=1 — what CI ships to Pages
npm run preview       # serves dist/ at http://localhost:4173/news-radar-dashboard/
```

## Data source

Live mode fetches `https://raw.githubusercontent.com/HsinTiger/news-radar/state/data/01_harvest/news_radar.db` (~4.5 MB) and opens it with sql.js. The DB is regenerated by the backend's `mac_compose` job; Alex pushes a fresh `state` branch commit every harvest cycle. The dashboard polls every 5 minutes — refresh is driven by a `setInterval` in `useNewsRadarDB`.

Tables read by the dashboard:
`news_items`, `drafts`, `platform_drafts`, `publish_log`, `engagement_stats`, `topic_weights`, `reflection_events`, `token_usage_daily`.

No writes — `useNewsRadarDB` only opens the DB `readOnly`.

**Falling back when the fetch fails.** Network error, GitHub outage, or bad DB → `useNewsRadarDB` exposes `{ loading: false, error, items: [] }`. Pages render an empty state rather than crashing.

## Deployment (GitHub Pages)

This repo is designed to deploy to `HsinTiger.github.io/news-radar-dashboard/` as project pages. Vite `base` is set to `/news-radar-dashboard/` in `vite.config.js`.

Push to `main` → the GitHub Actions workflow in `.github/workflows/deploy.yml` builds and publishes to the `gh-pages` branch.

See `docs/DEPLOY.md` for first-time setup (repo creation, enabling Pages).

## Why this design

- **sql.js in-browser** — no backend to maintain, no secrets in the dashboard, full schema visibility for free.
- **Polling, not websockets** — a harvest cycle is ~90 min; 5 min refresh is more than sufficient and costs us one GET per interval.
- **Mock-first** — `src/data/mockData.js` is the spec. Any change to `dbAdapter.js` must keep producing the same shape so pages stay agnostic to data source.
- **No write actions** — the dashboard is pure observation. All approvals, edits, retries happen in the backend CLI tools.
